# Get Active Orders

> `GET /orders`

Get the list of active orders for the requesting user. The user context is resolved based on the provided API key.

> **Note:** Rate limit: 1000 requests per minute.

---

## Request

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `symbol` | string | | Filter orders by trading pair symbol (e.g. `BTC/USD`). |
| `side` | string | | Filter by order side. Possible values: `buy`, `sell`. |
| `limit` | integer | | Maximum number of orders to return. |

### Header Parameters

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `X-Revx-Timestamp` | integer | ✅ Required | Current timestamp in Unix epoch milliseconds. Used to prevent replay attacks and construct the signature. |
| `X-Revx-Signature` | string | ✅ Required | The Ed25519 signature of the request. |

**Example `X-Revx-Timestamp`:** `1746007718237`

**Example `X-Revx-Signature`:**
```
2h/t5o8w+l5s+fjyFAOn/o7j4u5b7h4e+g4k4c8h7a2p6k0g7j1fiw0i2j3k9r0l3s8m5t6r+q1s+o3v/t4x8v5y+w1r+m2t/k3w/j4y+
```

> **Tip:** See [Authentication headers: Signing a request](#) for details on how to generate the signature.

## Response

### 200 – OK

**Body:** `object`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `data` | array of objects | ✅ Required | List of active orders. |

Each order object in `data` contains:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `venue_order_id` | string (uuid) | ✅ Required | ID of the order generated by the system. |
| `client_order_id` | string (uuid) | ✅ Required | Client order ID provided by the client. |
| `symbol` | string | ✅ Required | The trading pair symbol, e.g. `BTC/USD`. |
| `side` | string | ✅ Required | The direction of the order. Possible values: `buy`, `sell`. |
| `state` | string | ✅ Required | Current order state. Possible values: `pending_new`, `new`, `partially_filled`, `filled`, `cancelled`, `rejected`, `replaced`. |
| `type` | string | ✅ Required | The type of the order. |
| `created_at` | string (date-time) | ✅ Required | Timestamp when the order was created. |
| `updated_at` | string (date-time) | ✅ Required | Timestamp when the order was last updated. |
| `base_size` | string (decimal) | | Amount of base currency. |
| `quote_size` | string (decimal) | | Amount of quote currency. |
| `price` | string (decimal) | | The order limit price. |
| `filled_size` | string (decimal) | | Amount of base currency already filled. |
| `filled_value` | string (decimal) | | Value of the filled portion in quote currency. |
| `average_filled_price` | string (decimal) | | Average price at which the order has been filled so far. |
| `execution_instructions` | array of strings | | Execution instructions for the order. Possible values: `allow_taker`, `post_only`. |
| `time_in_force` | string | | Time-in-force policy. |
| `legs` | array | | Legs of the order, if applicable. |
