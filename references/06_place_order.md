# Place Order

> `POST /orders`

Place a new order. The user is resolved by the provided API key.

> **Note:** Rate limit: 1000 requests per minute.

---

## Request

### Header Parameters

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `X-Revx-Timestamp` | integer | ✅ Required | Current timestamp in Unix epoch milliseconds. Used to prevent replay attacks and construct the signature. |
| `X-Revx-Signature` | string | ✅ Required | The Ed25519 signature of the request. |

**Example `X-Revx-Timestamp`:** `1746007718237`

**Example `X-Revx-Signature`:**
```
2h/t5o8w+l5s+fjyFAOn/o7j4u5b7h4e+g4k4c8h7a2p6k0g7j1fiw0i2j3k9r0l3s8m5t6r+q1s+o3v/t4x8v5y+w1r+m2t/k3w/j4y+
```

> **Tip:** See [Authentication headers: Signing a request](#) for details on how to generate the signature.

### Example cURL

```bash
curl -i -X POST 'https://revx.revolut.com/api/1.0/orders' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -H 'X-Revx-API-Key: <API_KEY_VALUE>' \
  -d '{
    "client_order_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "symbol": "BTC/USD",
    "side": "sell",
    "order_configuration": {
      "limit": {
        "base_size": "0.1",
        "price": "50000.50",
        "execution_instructions": []
      }
    }
  }'
```

---

### Request Body

**Body:** `object`

Order placement details.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `client_order_id` | string (uuid) | ✅ Required | Unique identifier for idempotency. Example: `"3fa85f64-5717-4562-b3fc-2c963f66afa6"`. |
| `symbol` | string | ✅ Required | The trading pair symbol. Example: `"BTC-USD"`. |
| `side` | string | ✅ Required | The direction of the order. Possible values: `buy`, `sell`. |
| `order_configuration` | object | ✅ Required | Configuration for the order. Must contain exactly one of `limit` or `market`. |

#### `order_configuration.limit` — object

Limit order configuration. Must contain exactly one of `base_size` or `quote_size`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `base_size` | string (decimal) | | Amount of base currency. Example: `"0.1"`. |
| `quote_size` | string (decimal) | | Amount of quote currency. Example: `"0.1"`. |
| `price` | string (decimal) | ✅ Required | The limit price. Example: `"50000.50"`. |
| `execution_instructions` | array of strings | | List of execution instructions. Possible values: `allow_taker`, `post_only`. Default: `allow_taker`. Can be an empty array `[]` if no specific instructions are required. |

#### `order_configuration.market` — object

Market order configuration. Must contain exactly one of `base_size` or `quote_size`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `base_size` | string (decimal) | | Amount of base currency. Example: `"0.1"`. |
| `quote_size` | string (decimal) | | Amount of quote currency. Example: `"0.1"`. |

---

## Response

### 200 – OK

The successfully placed order details.

**Body:** `object`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `data` | object | ✅ Required | Wrapper object containing the order details. |
| `data.venue_order_id` | string (uuid) | ✅ Required | ID of the order generated by the system. |
| `data.client_order_id` | string (uuid) | ✅ Required | Client order ID provided by the client. |
| `data.state` | string | ✅ Required | Order state after submitting. Possible values: `pending_new`, `new`, `partially_filled`, `filled`, `cancelled`, `rejected`, `replaced`. |

**Example Response Body:**
```json
{
  "data": [
    {
      "venue_order_id": "7a52e92e-8639-4fe1-a0aa-68d3a2d523...",
      "client_order_id": "984a6d8e-2e96-495d-812f-2a4803f...",
      "state": "new"
    }
  ]
}
```
